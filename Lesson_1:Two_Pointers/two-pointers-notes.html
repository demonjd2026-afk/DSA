01 ‚Äî Foundation

The Core Idea

Instead of using a nested loop to check every pair (O(n¬≤)), we use two index variables that traverse the data structure ‚Äî either towards each other, in the same direction, or at different speeds ‚Äî to find the answer in a single pass O(n).

5 Core Variants

01
Variant 1
Opposite Ends
(Converging)
L starts at index 0, R starts at last index. They move toward each other until they meet. Works on sorted arrays.

Same Direction
(Sliding Window)
Both pointers start at 0 and move right. R expands the window; L shrinks it when a condition is violated. Works on unsorted data.

Variant 3
Fast & Slow
(Floyd's Cycle)
Slow moves 1 step, Fast moves 2 steps. If they meet, a cycle exists. Also finds the middle of a linked list.

Ready-to-Use Templates

// V1 ‚Äî Opposite Ends

def two_sum_sorted(arr, target): L, R = 0, len(arr) - 1 while L < R: curr_sum = arr[L] + arr[R] if curr_sum == target: return [L, R] elif curr_sum < target: L += 1 # need bigger ‚Üí move left right else: R -= 1 # need smaller ‚Üí move right left return []

// V2 ‚Äî Sliding Window

def longest_no_repeat(s): seen = {} L = 0 max_len = 0 for R in range(len(s)): if s[R] in seen and seen[s[R]] >= L: L = seen[s[R]] + 1 # shrink window from left seen[s[R]] = R max_len = max(max_len, R - L + 1) return max_len

// V3 ‚Äî Fast & Slow (Cycle Detection)

def has_cycle(head): slow, fast = head, head while fast and fast.next: slow = slow.next # 1 step fast = fast.next.next # 2 steps if slow == fast: return True # cycle found! return False

// V4 ‚Äî Partition / Write Pointer

def move_zeroes(nums): write = 0 # write pointer for read in range(len(nums)): if nums[read] != 0: nums[write] = nums[read] write += 1 while write < len(nums): nums[write] = 0 write += 1

06 ‚Äî Pro Tips

Things to Remember

üîÉ
Sort First if Needed
V1 (Opposite Ends) requires sorted input. If unsorted, sort it first O(n log n) ‚Äî still better than O(n¬≤).
‚ö†Ô∏è
Handle Duplicates
In 3Sum / pair problems, skip duplicate values after finding an answer: while L < R and arr[L] == arr[L+1]: L++
ü™ü
Window Validity
In sliding window, always track the condition that makes a window valid/invalid. Use a counter or hashmap to track this efficiently.
üê¢
Slow = Middle
When fast pointer reaches end, slow pointer is at middle. Useful for finding midpoint to reverse second half of list.
üìè
Window Size Formula
Current window size = R - L + 1. Current window sum/elements are tracked with a variable updated as L/R move.
üîÅ
Loop Condition
V1: while L < R  |  V2: for R in range(n)  |  V3: while fast and fast.next  |  V4: for read in range(n)
